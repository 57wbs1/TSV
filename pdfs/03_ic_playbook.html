What changes when you're the IC

Boarding SITREP — Transport flow

When a bus or flight is due to move your syndicate, the Transport sub-tab is the control panel. The flow has three phases: board, push, drop.

Phase 1 · Board

Open Calendar → Transport.
Tap the bus or flight card assigned to your syndicate.
A list of syndicates appears. Tap Board next to each syndicate as they physically step onto the vehicle. You can board multiple syndicates — a single bus often carries two or three.
If not everyone is on yet and you need to save progress, tap Save Progress. It records what you've boarded so far without closing the flow.
Once the vehicle is full and ready, tap Send SITREP. A 🚌 BOARDING COMPLETE message fires to the ops chat with the syndicate count.

Phase 2 · Push

When the vehicle departs, tap Pushing on the same card.
Optional: add remarks such as Departing hotel 0851H, ETA RTAF 0935H.
Send. An ON THE MOVE message fires with driver and timing info.

Phase 3 · Drop

On arrival, tap Dropped on the card.
The vehicle is freed for the next cycle. Its card returns to the idle state for the next syndicate to use.
Why all three. Each phase fires a distinct Telegram message. Command needs to know boarded, moving, and arrived as separate events — the gap between each tells them if something is going wrong mid-journey.

Ad-hoc SITREP from Home

When movement changes during the day — a bus diverts, your syndicate splits, half the group leaves a visit early — a SYN IC can push a fresh SITREP in seconds. The button is on Home so you don't have to drill into Tracker first.

Open the Home tab.
Tap 📤 Send SITREP near the top (only visible to ICs).
Choose which syndicates to include. The default is your own.
Optional remarks — be specific: Syn 3 split at Wat Pho, 8 pax returning with me, 6 pax onward to MBK w/ CPT Zhang.
Send. The SITREP lands in the ops chat immediately.
Regular officers as fallback. If for some reason you're unreachable, any officer in your syndicate can push the same ad-hoc SITREP from the Tracker list view. It goes to the same chat. You cover each other.

Incident Reports — the full lifecycle

Anything that isn't routine gets logged as an IR. Medical, security, missing persons, transport failures, schedule changes that leadership needs to know about. An IR has three states — NEW, UPDATE, and CLOSED — and every state transition posts a new Telegram message.

Creating a new IR

Open the SOP tab.
Tap + New Incident Report at the top.
Fill the form:
  • Nature. Pick from the dropdown (medical, security, transport, welfare, etc.).
  • Description. What happened, in one or two sentences.
  • When. Time of the incident — e.g. 270426 / 0900HRS.
  • Where. Location.
  • Course / Syn involved. Dropdown of syndicates.
  • NOK Informed. Y, N, or N/A.
Tick Send to Telegram (almost always yes).
Submit. The server assigns a sequential ID — IR01, IR02, IR03 — and a 🚨 IR NEW message lands in the command chat.

Posting an update

As the incident progresses, keep the feed current. Every update re-posts the full history so command always sees the complete thread in one message.

Open the IR from the SOP tab list.
Tap ➕ Add Update.
Type the update — e.g. Officer admitted to BNH Hospital, vitals stable, expected discharge 1800H.
Keep Send to Telegram ticked.
Submit. A 📝 IR UPDATE #n message fires with the new update plus a copy of the original and all prior updates.

Closing out

When the incident is resolved:

Open the IR. Tap ➕ Add Update.
Write a final update describing the resolution.
Tick the Close Out checkbox.
Submit. A ✅ IR CLOSED message fires. The IR stays visible and searchable in the SOP tab for the rest of the trip, but no further updates can be added.
If Telegram fails to send. A toast will say Logged but Telegram failed — resend later. Open the IR event and tap the 📤 Resend button next to it. That re-sends the exact message without creating a new event.

Deleting an IR

If an IR was created in error — wrong details, duplicate entry, test run — only admins can delete it. You as SYN IC cannot. If you need one deleted, write to the admin through Settings → Report an Issue with the IR number and reason.

Three rules for ICs

  1. Board → Push → Drop. Every transport cycle, every bus, every flight. Three taps. Don't skip the middle one.
  2. If the movement changes, fire a SITREP. From Home, one tap, 30 seconds. Command hates surprises; they love timely updates.
  3. Open an IR earlier than you think. It is cheap to file and easy to close. A fever that turned out to be nothing is a closed IR — it is not an embarrassment. An unreported fever that turned into a hospital admission is an investigation.
